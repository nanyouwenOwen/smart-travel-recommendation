# 安全、限流、性能与恢复基线

- 安全：现有集成测试覆盖匿名/越权、Token、输入边界、咨询注入/隐私、SSE 归属；前端 `npm audit --audit-level=high` 为发布门，镜像使用非 root 用户且不内置 secret。
- 限流/恢复：行程与咨询测试覆盖用户 RPM/并发、AI 超时/429/5xx、任务恢复、SSE 重放/取消；实时数据覆盖缓存、归属和受控降级。Compose smoke 重启后端并等待健康恢复。
- 性能：`scripts/perf.sh` 使用固定 k6 0.57.0，10 VU/30 秒同时请求健康代理和前端首页；共享 GitHub Runner 的门槛为错误率 `<1%`、p95 `<1000ms`。该数值仅是预热后小型演示基线，不是生产 SLA。
- 容器扫描：当前候选以依赖审计、最小运行镜像和 CI 构建为门；生产发布前仍建议在组织环境增加 Trivy/registry 持续扫描。
